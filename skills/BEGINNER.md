## What You'll Learn

By completing beginner projects, you'll understand:

1. **Outcome-Driven Requests**
   - Say WHAT you want, not HOW to do it
   - Let Claude choose the tools
   - Get better results faster

2. **Natural Language Works**
   - No syntax to learn
   - No commands to memorize
   - Just describe your goal

3. **Trust and Delegation**
   - Claude analyzes first
   - Proposes approach
   - You approve
   - Claude executes

4. **Immediate Value**
   - Work on real files
   - See real results
   - Use them right away

## Common Mistakes (and How to Avoid Them)

### Mistake 1: Specifying Tools
❌ "Use Glob to find .jpg files"
✅ "Show me all my photos"

**Why it's wrong**: You're doing Claude's job. Let Claude choose the best approach.

### Mistake 2: Being Too Vague
❌ "Help me with files"
✅ "Organize my Downloads folder by file type and date"

**Why it's wrong**: Claude needs to know WHAT you want to accomplish.

### Mistake 3: Not Trusting
❌ Interrupting Claude's work to micromanage
✅ Let Claude finish, review the result, then adjust if needed

**Why it's wrong**: Claude works best with full context and clear goals.

### Mistake 4: Fake Practice
❌ Organizing sample files you'll never use
✅ Organizing your actual Downloads folder

**Why it's wrong**: You learn best by solving real problems with real value.
